# Liveness Detection — Prototype

เว็บเดโมตรวจความมีชีวิต (liveness) บนเบราว์เซอร์ **โดยประมวลผลบนอุปกรณ์** — วิดีโอจากกล้องไม่ถูกส่งไปเซิร์ฟเวอร์ของแอปนี้ ใช้ [MediaPipe Face Landmarker](https://ai.google.dev/edge/mediapipe/solutions/vision/face_landmarker) (`@mediapipe/tasks-vision`) ผ่าน WebGPU/WebGL ตามที่รันไทม์รองรับ

## คุณสมบัติหลัก

- **ชุดคำสั่งสุ่ม**: กระพริบตา, ยิ้ม, หันหน้าซ้าย/ขวา — ลำดับสับเปลี่ยนในแต่ละเซสชัน
- **ตรวจกระพริบ**: ใช้ทั้ง Eye Aspect Ratio (EAR) และ Face Blendshapes จากโมเดล เพื่อลดกรณีรูปสองมิติ
- **ตรวจ micro-motion**: วัดการเปลี่ยนแปลงความสว่างบริเวณใบหน้า — ถ้าใบหน้านิ่งเกินเกณฑ์นานเกินไป เซสชันจะล้มเหลว (ช่วยกันหน้าจอนิ่ง/รูปถือ)
- **ทดสอบจากไฟล์วิดีโอ**: เมื่อไม่มีกล้องหรือต้องการทดสอบออฟไลน์

## ข้อกำหนด

- [Node.js](https://nodejs.org/) เวอร์ชันที่รองรับ ES modules (แนะนำ LTS)
- เบราว์เซอร์สมัยใหม่ที่รองรับ `getUserMedia` และ runtime ของ MediaPipe Tasks

## การติดตั้งและรัน

```bash
npm install
npm run dev
```

จากนั้นเปิด **`https://localhost:5173`** (โปรเจกต์ใช้ Vite พร้อม `@vitejs/plugin-basic-ssl` เพื่อให้กล้องทำงานในโหมด secure context)

- **ครั้งแรก** เบราว์เซอร์อาจเตือนเรื่องใบรับรองชั่วคราว — ให้ยอมรับตามขั้นตอนของเบราว์เซอร์
- **กล้อง**: ควรเข้าผ่าน `https://localhost` — การเปิดผ่าน `http://` หรือ IP ในเครือข่ายอาจทำให้กล้องถูกบล็อก
- ถ้าใช้กล้องไม่ได้ ใช้ปุ่ม **ทดสอบจากไฟล์วิดีโอ** แทน

## สคริปต์

| คำสั่ง | คำอธิบาย |
|--------|----------|
| `npm run dev` | เซิร์ฟเวอร์พัฒนา (HTTPS, พอร์ต 5173) |
| `npm run build` | build สำหรับ production |
| `npm run preview` | พรีวิว build หลัง `build` |

## โครงสร้างโปรเจกต์ (ย่อ)

- `index.html` — หน้า UI (ภาษาไทย)
- `src/main.js` — โหลดโมเดล, กล้อง/ไฟล์, logic ชาเลนจ์และ anti-spoof
- `src/style.css` — สไตล์
- `vite.config.js` — HTTPS dev server, `host: true`

โมเดล Face Landmarker โหลดจาก URL ของ Google (ดูค่าคงที่ `MODEL_URL` ใน `src/main.js`)

## ข้อจำกัด (สำคัญ)

นี่เป็น **ต้นแบบทางเทคนิค** ไม่ใช่ระบบยืนยันตัวตนระดับ production

- วิดีโอที่เล่นบนหน้าจอหรือ deepfake บางแบบอาจหลอกได้บ้าง
- การใช้งานจริงควรมี **challenge จากเซิร์ฟเวอร์**, นโยบายอัตราการลองใหม่, และโมเดลตรวจ spoof เพิ่มเติมตามความเสี่ยง
